Description

In our hospital, there are more and more patients. The receptionist used to write down the visits on sticky notes. Unfortunately, the sticky notes get lost or taken away by doctors. The hospital director thinks that at the beginning a simple program for the receptionist will be sufficient.

Let's make our program with the help of a REST service. To create REST service we will create Spring project using Spring Boot . 

The following dependencies should be pasted to build.gradle file (in our study project they are already included): 

spring-boot-starter - Core starter, including auto-configuration support, logging and YAML;

spring-boot-starter-web - Starter for building web, including RESTful, applications using Spring MVC. Uses Tomcat as the default embedded container

In this stage, you need to create 3 endpoints that will return the information about appointments in JSON format. At this stage to store information about appointments you  can use interface Map<K,V>. In next stages we will store inforamtion about appointments in the database. 

Objectives

Implement the following endpoints:  

1)   Implement  /setAppointment  endpoint that handles POST requests to create new appointment and returns information about the appointment.

A request should contain the following body:

{
    "doctorName": "Name of the doctor",
    "patientName": "Name of the patient",
    "date": "date of visite : 2021-12-01 22:00"

}

Take these variables and store them for the appointment. 

Response body should be JSON object  as following example:

{ "idApp": 1, "doctorName": "Pamela Upperson ", "patientName": "John Galt", "date": "2021-12-01 22:00" }

where "idApp" is unique id assigned to the appointment in Number format,

doctorName is name of the doctor which was in the request body and stored in the appointment with mentioned idApp in String format;

patientName  is name of the patient which was in the request body and stored in the appointment with mentioned idApp in String format; date  is the date of visit  which was in the request body and stored in the appointment with mentioned idApp in String format;

The endpoint should return Response Status Code 200;

2)    Implement /deleteAppointment/?id={N}  endpoint that handles DELETE requests to delete appointment and returns the information about the appointment deleted.

To delete an appointment please use id  assigned to the appointement when it was created. Use the id as a request parameter.

If appointment with requested id exists , delete the appointment and  return Response Status Code 200;

Response body should be JSON object  as following example:

{ "idApp": 1, "doctorName": "Pamela Upperson ", "patientName": "John Galt", "date": "2021-12-01 22:00" }

where "idApp" is unique id assigned to the appointment in Number format,

doctorName is name of the doctor which was in the request body and stored in the appointment with mentioned idApp in String format;

patientName  is name of the patient which was in the request body and stored in the appointment with mentioned idApp in String format; date  is the date of visit  which was in the request body and stored in the appointment with mentioned idApp in String format;

if appointment  with requested id  does not exists or was deleted then return error message "The appointment was already cancelled or does not exist!" and Response Status Code 400;

3)    Implement /appointments endpoint that handles GET requests and returns the information about all appointments. The endpoint do not accept any body or parameters and return respons with all created  appointments.

The endpoint should return Response Status Code 200;

The response as a JSON objects should contain unique number in String format as a name and value  a JSON object which contains the following name/value pairs: 

 "idApp":  appointment number in Number format,

  "doctorName": doctor name taken from the appoinment in String format,

   "patientName": patient name taken from the appoinment in String format,

  "date": date of appointment in  the format "yyyy-MM-dd HH:mm"

 Example of the response as a JSON object :


{
    "1": {
        "idApp": 1,
        "doctorName": "Pamela Upperson",
        "patientName": "John Galt",
        "date": "2021-12-01 11:00"
    },
    "2": {
        "idApp": 2,
        "doctorName": "Pamela Upperson",
        "patientName": "Phill good",
        "date": "2021-12-01 12:00"
    }
    "3": {
        "idApp": 3,
        "doctorName": "Phill good",
        "patientName": "Phill good",
        "date": "2021-12-01 13:00"
    }
}

 

Request and response entity-header Content-Type  of all the API endpoints should be equal to application/json  .
